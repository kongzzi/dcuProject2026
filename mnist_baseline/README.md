# 손글씨 숫자 인식 그림판 — 베이스라인 (초급 ★★ 자체 모델형)

**직접 학습한 PyTorch 모델**을 FastAPI로 서빙하고, Streamlit 그림판에서 실시간 인식하는 프로젝트입니다.
베이스라인이 이미 동작하므로, 여러분의 과제는 **정확도 올리기 + 실험 기록 + 데모 완성**입니다.

```
[Streamlit 그림판 (app.py)] → [FastAPI 서버 (server.py)] → [내가 학습한 model.pt]
                                      ↑
                      [train_mnist.ipynb 에서 학습 후 저장]
```

## 1. 설치

```bash
pip install -r requirements.txt
```

## 2. 진행 순서

```bash
# 1) 노트북 실행 → 학습 → model.pt 저장 (전체 셀 위에서부터 실행)
jupyter notebook train_mnist.ipynb

# 2) 터미널 1 — 서빙 서버 (model.pt 가 있어야 뜹니다)
uvicorn server:app --reload --port 8000

# 3) 터미널 2 — 그림판 UI
streamlit run app.py
```

브라우저 그림판에 숫자를 그리고 [인식하기] → 예측 숫자와 확률 그래프가 나오면 성공.

## 3. 파일 구성

| 파일 | 역할 |
|---|---|
| `train_mnist.ipynb` | 학습 → 평가 → `model.pt` 저장. **TODO와 실험 기록 표가 여기 있음** |
| `model.py` | 모델 구조(SmallCNN) + 정규화 상수. 노트북과 서버가 **공유** |
| `server.py` | `model.pt` 로드, `/predict` (28x28 픽셀 → 숫자 + 확률 10개) |
| `app.py` | 그림판 UI. 그림을 28x28로 줄여 서버에 전송 |

## 4. 과제 흐름 (5일 기준)

| 일차 | 목표 | 완성 기준 |
|---|---|---|
| 1 | 환경 설치, 베이스라인 노트북 완주 | `model.pt` 생성, baseline 정확도 기록 |
| 2 | TODO 실험 (epoch·lr·Dropout·증강) | 정확도 97%+ 및 실험 기록 2건 이상 |
| 3 | 서버 + 그림판 연결 | 그림 → 예측이 화면에 표시 |
| 4 | 심화 TODO(확신도 낮음 처리 등) + 오류 처리 점검, 버퍼 | 빈 캔버스·서버 꺼짐에도 앱이 안 죽음 |
| 5 | PPT(실험 기록 표 포함) + 리허설 + 시연 영상 | 30초 라이브 데모 성공 |

## 5. 알아두면 좋은 것

- **왜 내 글씨는 가끔 틀리나요?** MNIST 글씨는 중앙 정렬·일정한 크기입니다. 구석에 작게 그리면 틀리기 쉬워요.
  → 크게, 가운데에 그리세요. (심화: 입력 그림을 중앙 정렬하는 전처리를 추가하면 발표 소재가 됩니다)
- **모델 구조를 바꿨는데 서버가 에러**: `model.py` 수정 → 노트북에서 재학습·재저장 → 서버 재시작 순서를 지키세요.
- **발표 팁**: "baseline 97.0% → 실험 3번으로 98.5%" 같은 실험 기록 표 + 청중이 직접 그려보는 라이브 데모 조합이 가장 강력합니다.
